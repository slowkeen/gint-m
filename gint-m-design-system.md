## 1. ДИЗАЙН-СИСТЕМА С НУЛЯ

Поскольку брендбука нет и палитру строим с нуля — это одновременно и вызов, и возможность. Я предлагаю сразу создать систему, которая будет работать на современное позиционирование.

### 1.1 Концепция: «Архитектурная точность»

Гинт-М — генподрядчик с 20+ годами опыта, работающий с международными клиентами (Microsoft, SAP, Kaspersky). Дизайн должен транслировать:

- **Надёжность** — через стабильную, сдержанную палитру
- **Точность** — через чёткую типографику и геометричную сетку
- **Масштаб** — через крупные фотографии и обилие воздуха
- **Современность** — через анимации и актуальные паттерны

Ориентир — Pridex, но без копирования: Pridex позиционируется как «технологичные отделочные работы», Гинт-М — как «надёжный генподрядчик полного цикла». Тональность чуть теплее и основательнее.

### 1.2 Цветовая палитра

Существующие фирменные цвета не вписываются в современную минималистичную эстетику. Палитра строится с нуля, с ориентиром на сдержанный, премиальный визуал (как у Pridex).

#### Вариант A: «Тёплый индустриальный» (мой основной вариант)

| Роль | Цвет | HEX | Применение |
|---|---|---|---|
| Тёмный фон | Антрацит | #1C1C1E | Первый экран, футер, акцентные секции |
| Светлый фон | Тёплый белый | #FAF9F7 | Основной контент |
| Вторичный фон | Тёплый серый | #F0EEEA | Чередование секций, карточки |
| Основной текст | Почти-чёрный | #1A1A1A | Заголовки, основной текст |
| Вторичный текст | Средний серый | #6B7280 | Подписи, мета-информация |
| Акцент | Терракот | #C05A3C | кнопки с призывом к действию, эффекты при наведении, выделения |
| Акцент при наведении | Тёмный терракот | #A04830 | Состояние нажатия |
| Линии/разделители | Светлый серый | #E5E2DD | Бордеры, разделители |

**Почему терракот:** тёплый, ассоциируется со строительством и материалами (кирпич, глина), выделяется на фоне референсов; у Pridex нейтральная палитра, поэтому терракот не выглядит «корпоративно-холодным» как синий.

|---|---|---|---|

**Когда выбрать:** если заказчик хочет максимально приблизиться к эстетике Pridex (тёмный + нейтральный). Более «корпоративный» вариант.

| Роль | Цвет | HEX | Применение |
|---|---|---|---|

**Когда выбрать:** если хотят максимально премиальное, «архитектурное бюро» ощущение. Подходит для позиционирования в верхнем сегменте.

### 1.3 Типографика

#### Предлагаюая пара шрифтов

**Заголовки: Outfit**
- Геометрический, современный, но с мягкостью
- Отлично работает в крупных кеглях (заголовки первого экрана 64–80px)
- Бесплатный (Google Fonts), кириллица поддерживается
- Альтернативы: Manrope, Unbounded, TT Norms Pro

**Текст: Source Sans 3 (бывш. Source Sans Pro)**
- Профессиональный, отличная читаемость
- Хорошо работает в 16–18px для основного текста
- Кириллица поддерживается
- Альтернативы: Golos Text, Nunito Sans

**Почему не Inter/Roboto:** слишком распространены, не создают запоминающегося визуального впечатления. Outfit даёт узнаваемый характер, оставаясь при этом деловым.

#### Иерархия размеров

|---|---|---|---|
| Заголовок первого экрана (H1) | 72px / 80px | 40px / 48px | 700 (Bold) |
| Секция H2 | 48px | 32px | 600 (SemiBold) |

## 2. ПОРТФОЛИО — КЛЮЧЕВОЙ РАЗДЕЛ

Поскольку портфолио — приоритет, я детализировал предложения для этого раздела.

### 2.1 Каталог проектов (/projects/)

**Верх страницы:**
- Зона первого экрана: крупный заголовок «Проекты» + счётчик «500+ реализованных объектов»
- Фоновое фото лучшего проекта с затемнением

**Блок избранных проектов (новый):**
- 2–3 флагманских проекта крупно (на всю ширину или 50/50)
- Каждый: крупное фото + название + год + площадь + кнопка «Смотреть»
- Это задаёт «планку качества» для всего портфолио

**Фильтры:**
Визуально выделенные теги с количеством:
`Все (500+) | Офисные (120) | Спортивные (15) | Капстрой (30) | Ритейл (45) | Жилая (20)`

Добавить фильтр по площади (как у Pridex):
`до 500 м² | 500–1000 | 1000–5000 | 5000–10 000 | 10 000+`

**Сетка:**
- «Кирпичная» сетка (разные размеры карточек) — выглядит более «архитектурно»
- Или строгая сетка 3 колонки с эффектами при наведении
- Загрузка: «Показать ещё» вместо пагинации

**Блок клиентов внизу:**
Логотипы с переходом на фильтр по клиенту. Сейчас этот фильтр скрыт — предлагаю сделать его первоклассным элементом.

### 2.2 Страница проекта (детальная)

**Структура сверху вниз:**

```
1. ПЕРВЫЙ ЭКРАН (полноэкранное фото)
   ├── Название проекта (Outfit Bold, белый)
   ├── Категория-тег
   └── Индикатор прокрутки ↓

2. МЕТА-БЛОК (фиксированная боковая колонка на больших экранах / блок на мобильных)
   ├── Логотип клиента
   ├── Площадь: 5 200 м²
   ├── Адрес: Москва, ...
   ├── Вид работ: отделочные работы, инженерные системы
   ├── Срок реализации: 8 месяцев
   └── Год: 2024

3. ОПИСАНИЕ
   ├── Задача (2–3 предложения)
   ├── Решение (2–3 предложения)
   ├── Результат (1–2 предложения)
   └── [Цитата заказчика, если есть]

4. ГАЛЕРЕЯ
   ├── Крупные фото (полная ширина или раскладка в 2 колонки)
   ├── Просмотр в увеличении при клике
   └── До/После (если применимо)

5. ТЕХНИЧЕСКИЕ ДЕТАЛИ (опционально)
   └── Инфографика: этажи, рабочие места, инженерные системы

6. ПОХОЖИЕ ПРОЕКТЫ
   └── 3 карточки из той же категории

7. Призыв к действию
   └── «Хотите обсудить похожий проект?» + кнопка

8. НАВИГАЦИЯ
   └── ← Предыдущий | Следующий →
```

**Ключевое отличие от текущего сайта:** структура «Задача → Решение → Результат» вместо монолитного текста. Это формат кейса, который аудитория корпоративных клиентов привыкла читать.

### 2.3 Мои предложения по фотоконтенту

Качество фотографий — самый важный фактор для раздела портфолио. Вот что я предлагаю:

**Обязательно:**
- Профессиональная съёмка каждого ключевого проекта (фотограф архитектурного/интерьерного направления)
- Единый стиль обработки: нейтральные тона, естественное освещение, выправленные вертикали
- Минимум 8–12 фото на проект: общие планы, детали, разные ракурсы
- Фото людей в пространстве (не пустые интерьеры) — создаёт ощущение «живого» проекта

**Технические требования:**
- Формат: WebP для сайта, JPEG высокого качества для архива
- Соотношение: 4:3 для каталога, 16:9 для первого экрана, 1:1 для деталей
- Минимальное разрешение: 2400px по длинной стороне
- Отложенная загрузка для галерей

**Что делать с проектами без качественных фото:**
- Не включать в блок избранных проектов
- Использовать карточку меньшего размера в каталоге
- Рассмотреть 3D-визуализации как замену (если есть)

За 5 секунд посетитель понимает: кто (генподрядчик), сколько опыта (20+ лет), масштаб (500+ проектов), что делать (обсудить или смотреть).

### 3.2 Анимации и микровзаимодействия

Анимации должны быть сдержанными и элегантными — без перегрузки, не влияющими на скорость загрузки. Баланс: сайт «живой», но не превращается в шоурил.

**Загрузка страницы (первый экран):**
1. Фон появляется мгновенно
2. Логотип — плавное появление (задержка 0.3s)
3. Заголовок — подъём вверх + плавное появление (0.5s)
4. Цифры — поочерёдное появление слева направо (0.7s–1.2s)
5. Кнопки — плавное появление (1.3s)

**Скролл:**
- Секции появляются с плавным подъёмом (через IntersectionObserver — наблюдатель пересечения)
- Счётчики цифр анимируются при появлении в зоне видимости
- Логотипы клиентов: из ч/б в цвет при прокрутке
- Лёгкий параллакс для фоновых изображений (не более 10% смещения)

**Эффекты при наведении:**
- Карточки проектов: увеличение (масштаб 1.03) + плашка + текст
- Кнопки: сдвиг цвета + лёгкое смещение
- Пункты меню: анимация подчёркивания (слева направо)
- Ссылки: плавная смена цвета 0.2s

**Чего избегать:**
- Тяжёлых анимаций, блокирующих взаимодействие
- Параллакс на мобильных (по производительности — избегать)
- Анимаций, которые воспроизводятся повторно при каждом скролле

### 3.3 Тон визуальной коммуникации

| Аспект | Текущий сайт | Как должно быть |
|---|---|---|
| Общее ощущение | «Корпоративный сайт 2015» | «Современное архитектурное бюро» |
| Фотографии | Разнородные, разного качества | Единый стиль, профессиональная съёмка |
| Типографика | PT Sans, мелкий кегль | Outfit + Source Sans, крупные заголовки |

| Анимации | Отсутствуют | Плавные, элегантные, сдержанные |

## 4. ПЛАТФОРМА: WORDPRESS (решение принято)

Перенос с UMI.CMS на WordPress. UMI устарел, дорого обслуживать, сложно найти специалиста. WordPress дешевле, нет подписок, интуитивное управление.

### Мои предложения по реализации на WordPress

**Тема:** индивидуальная (не готовый шаблон). Для имиджевого сайта генподрядчика готовая тема будет ограничивать дизайн. Разработка на базе базового шаблона Starter Theme (Sage, Underscores) с индивидуальными шаблонами.

**Конструктор страниц:** ACF (Advanced Custom Fields) Pro + индивидуальные Gutenberg-блоки. Это даёт:
- Гибкие шаблоны для проектов (разные сетки галерей)
- Поля для точки фокуса (focal_point), отметки избранного (is_featured), площади, срока реализации
- Простое управление для сисадмина

**Для проектов:**
- Пользовательский тип записи «Проекты» с таксономиями: тип (офисные, спортивные и т.д.), площадь, год
- ACF-поля: галерея с сортировкой, точка фокуса (focal_point) для каждого фото, отметка избранного (is_featured)
- Сортировка перетаскиванием

**Для новостей:**
- Стандартный тип записи с категориями
- Произвольная сортировка (текущая система управления сайтом не позволяет менять порядок — это одна из главных проблем)

**Оптимизация:**
- Отложенная загрузка изображений
- WebP с резервом на JPEG
- Отложенная загрузка скриптов и стилей
- Цель: скорость загрузки 1–1.3 сек
- Адаптивные изображения через `srcset`

**Безопасность:**
- SSL-сертификат
- Нестандартный адрес админки
- Регулярные бэкапы
- Ограничение попыток входа
