# Гинт-М — Детальные рекомендации по улучшению сайта
## С сохранением текущей структуры, ориентиром на Pridex и анализом 6 референсов и 3 конкурентов (Конте, TSM, Smart Company)

---

## Вводная часть

Я провёл анализ на основе: текущей структуры сайта Гинт-М, структуры сайта Pridex (как дизайн-ориентира), анализа 6 референсов и 3 конкурентов (Конте, TSM, Smart Company) и существующего документа с рекомендациями по редизайну.

**Ключевой принцип:** структура меню сохраняется (О компании → Новости → Услуги → Проекты → Награды → Контакты), но наполнение, компоновка блоков и визуальная подача кардинально обновляются. Платформа — WordPress (индивидуальная тема).

## 1. ГЛАВНАЯ СТРАНИЦА

### Что сейчас
Полноэкранный слайдер с проектами, слоган «Интерьер, вдохновляющий на успех», боковое меню. Нет цифр, нет призывов к действию, нет блока клиентов или услуг.

### Мои предложения по изменениям

#### 1.1 Первый экран: карусель + видео + цифры

**Предлагаю:** Сохранить принцип карусели, но кардинально обновить: каждый слайд — полноэкранное фото/видео проекта с наложением ключевых цифр и кнопок с призывом к действию. Первый слайд — видео (есть профессиональное видео умного офиса Кит Мед, планируется съёмка на новых объектах). Остальные слайды — лучшие проекты.

**Обоснование:** Pridex и референсы дополняют визуал цифрами и интерактивными элементами. Анимированные счётчики (анимация счётчиков при прокрутке) — один из самых эффективных приёмов на имиджевых сайтах строительных компаний. Совмещение карусели с анимированными счётчиками даёт и сильный визуальный эффект, и информативность.

**Предлагаемые цифры для первого экрана (уточнить реальные):**
- 27 лет на рынке (с 2001 года)
- 500+ реализованных проектов
- 150 000+ кв.м построено
- ISO 9001 / ISO 45001

**Кнопки:** «Обсудить проект» (основная) + «Смотреть портфолио» (вторичная/прозрачная)

#### 1.2 Блок «О компании» — краткое позиционирование

**Предлагаю:** Добавить компактный блок из 2–3 предложений + 4 карточки с иконками (генподряд, проектирование, инженерные системы, эксплуатация).

**Обоснование:** У Pridex на главной есть краткий блок «полный спектр услуг» с конкретными цифрами. У Гинт-М на главной вообще нет текста о том, чем компания занимается. Это критический недостаток: показатель отказов растёт, когда посетитель не находит ответ на вопрос «что здесь?» за первый скролл.

#### 1.3 Блок услуг

**Предлагаю:** 3 карточки (Проектирование, Строительство, Эксплуатация) с иконками, кратким описанием и ссылками на детальные страницы. Эффект при наведении.

**Обоснование:** Pridex выносит 5 направлений на главную с быстрыми ссылками. Это стандартный паттерн для строительных компаний, работающих с корпоративными заказчиками: показать широту компетенций, не заставляя кликать вглубь. Из референсов этот подход используют Baseform (карточки услуг) и Абис (карточки «Мы обеспечиваем»).

#### 1.4 Блок проектов — горизонтальная прокрутка

**Предлагаю:** Карусель из 6–8 лучших проектов с крупными фото, названием, годом, площадью. Кнопка «Все проекты».

**Обоснование:** Pridex использует карусель избранных проектов (HeadHunter, Lamoda, World Class). Это визуальное «социальное доказательство»: крупные имена клиентов вызывают доверие. У Гинт-М есть проекты для Microsoft, SAP, Kaspersky — их нужно выставлять на витрину.

#### 1.5 Блок клиентов — логотипы

**Предлагаю:** Бегущая лента или сетка логотипов ключевых клиентов.

**Обоснование:** У Pridex на главной есть сильные социальные доказательства: цифры и блок клиентов. У Абис — «благодарности 10 компаний». У Гинт-М блок клиентов на главной уже есть, но его стоит усилить: выделить ключевых заказчиков и показать их в отдельном, легко считываемом блоке. Такой блок — один из самых сильных аргументов доверия для корпоративных клиентов.

#### 1.6 Блок наград

**Предлагаю:** Компактный блок с 2–3 главными наградами (Best Office Awards и др.).

**Обоснование:** Ни один из проанализированных референсов не выносит награды на главную. Это даёт Гинт-М конкурентное преимущество: социальное доказательство через третью сторону (жюри) сильнее, чем самопрезентация.

#### 1.7 Блок с формой и призывом к действию

**Предлагаю:** Финальный блок перед футером: «Обсудим ваш проект?» + форма (имя, телефон, электронная почта, сообщение).

**Обоснование:** Сейчас призыв к действию есть только на странице контактов. Это значит, что на главной, услугах, проектах — нулевая конверсия без перехода на отдельную страницу. Pridex ставит кнопку «Получить консультацию» на каждую ключевую страницу. Из 6 референсов (без учёта 3 конкурентов) у 5 есть форма или призыв к действию на главной.

#### 1.8 Блок направлений / типов объектов

**Предлагаю:** На главной показать 6–8 типовых направлений (офисы, медицинские объекты, образование, торговые центры, гостиницы, жилые комплексы и т.д.) с быстрыми ссылками на проекты/фильтры.

**Обоснование:** У Smart Company навигация по направлениям — один из ключевых смысловых блоков. Это помогает посетителю быстро понять, «про него» ли компания, и сразу перейти в релевантные кейсы.

#### 1.9 География и охват работ

**Предлагаю:** Отдельный блок «Работаем по РФ и СНГ» + список ключевых городов/регионов или компактная карта охвата.

**Обоснование:** Smart Company демонстрирует федеральный масштаб и это усиливает доверие. Для корпоративных заказчиков география и масштабы — важный фактор выбора подрядчика.

## 2. О КОМПАНИИ (/about/)

### Что сейчас
Текст о компании, принципы в виде буллетов, логотипы клиентов (15 шт.), подменю с 4 подразделами (структура, качество, безопасность, экология).

### Мои предложения по изменениям

#### 2.1 Хронология истории компании

**Предлагаю:** Добавить визуальную хронологию: 2001 (основание) → ключевые вехи → 2025.

**Обоснование:** Pridex показывает «Pridex в цифрах» как инфографику. Интерьерно имеет подробную хронологию (2015–2024). У Гинт-М есть 24 года истории — это огромный актив доверия, который сейчас никак не визуализирован. Для корпоративных заказчиков длительность работы на рынке — один из главных факторов выбора подрядчика.

#### 2.2 Блок цифр

**Предлагаю:** Крупная инфографика: годы работы, количество проектов, построенная площадь, размер команды.

**Обоснование:** Pridex использует инфографику с цифрами (350+ проектов, 20+ команд, 4000+ специалистов, 80% повторных обращений). Гинт-М не показывает ни одной цифры на странице «О компании». Это упущенная возможность: цифры — самый быстрый способ передать масштаб.

#### 2.3 Команда с фото

**Предлагаю:** Карточки ключевых сотрудников (управляющий партнёр, директора департаментов) с фото, должностью, кратким описанием опыта.

**Обоснование:** Pridex показывает 10 карточек с фото и кнопкой «Подробнее». У Гинт-М есть детальная оргструктура (6 направлений), но нет ни одного лица. В строительной отрасли личное знакомство с руководством — важный фактор доверия.

#### 2.4 Подразделы — из табов в якорную навигацию

**Предлагаю:** Система качества, Безопасность труда, Экология — превратить в секции на одной длинной странице с якорной навигацией (как у Pridex на странице /about).

**Обоснование:** Текущие подстраницы (quality-control, health-and-safety, ecology) — это контент на 2–3 абзаца каждый. Отдельные страницы для такого объёма избыточны и создают ощущение «пустых» разделов. Якорная навигация (как у Pridex: Кто мы / Миссия / Команда / Цифры / КСО) позволяет показать всю глубину на одной странице без потери структуры.

#### 2.5 Миссия и ценности — визуальные карточки вместо буллетов

**Предлагаю:** Переоформить принципы из текстовых буллетов в визуальные карточки с иконками.

**Обоснование:** Pridex оформляет ценности как отдельные блоки (6 ценностей на странице карьеры). Baseform использует карточки для преимуществ. Текстовые буллеты выглядят «по-корпоративному скучно» и не запоминаются.

#### 2.6 Сертификаты и стандарты качества

**Предлагаю:** Выделить отдельный блок с сертификатами и стандартами (например, ISO/ГОСТ, допуски, охрана труда) + короткие пояснения, что это дает заказчику.

**Обоснование:** TSM явно показывает ISO 9001:2015 — это сигнал зрелости процессов и контроля качества. Для корпоративных тендеров это критичный аргумент.

## 3. УСЛУГИ (/services/)

### Что сейчас
3 карточки услуг с фото и длинными списками буллетов. Детальные страницы — сплошной текст.

### Мои предложения по изменениям

#### 3.1 Визуальная схема процесса работы

**Предлагаю:** Добавить блок «Этапы работы» — горизонтальная или вертикальная визуальная цепочка: Бриф → Проектирование → Согласование → Строительство → Сдача → Гарантия.

**Обоснование:** Из 6 референсов (без учёта 3 конкурентов) 4 имеют блок «Этапы работы» (Baseform, Aetheris, Fonbureau, Абис). У Pridex этапы описаны в каждой услуге (Подготовка → Проектирование → Строительство → Гарантийное обслуживание). У Гинт-М этапы упоминаются в разделе «Система качества» (проектирование → согласование → производство → логистика → сдача → гарантия → эксплуатация), но они визуально не оформлены и спрятаны от глаз посетителя. Визуализация процесса снимает главный страх корпоративного клиента: «а как будет проходить работа?»

#### 3.2 Переработка карточек услуг

**Предлагаю:** Каждая услуга — иконка + краткое описание (2–3 строки) + кнопка «Подробнее». Длинные списки работ убрать на детальные страницы.

**Обоснование:** Сейчас карточки перегружены буллетами (механические работы, электротехнические работы, и т.д.). Посетитель видит стену текста и не считывает главное. Pridex подаёт 5 направлений лаконично, а детали раскрывает при переходе.

#### 3.3 Детальные страницы услуг: добавить примеры проектов и блок вопросов-ответов

**Предлагаю:** На каждой странице услуги (проектирование, строительство, эксплуатация) добавить: 3–4 релевантных проекта из портфолио, мини-блок «вопросы и ответы» (3–5 вопросов), кнопку обращения.

**Обоснование:** Сейчас детальные страницы — это только текст. Нет визуальных подтверждений компетенций. Aetheris показывает раздельные блоки «вопросы и ответы» для клиентов и дизайнеров. Fonbureau и Baseform дают этапы на каждой услуге. Проекты на странице услуги — это «доказательство в действии».

#### 3.4 Блок преимуществ

**Предлагаю:** Добавить 4 преимущества с иконками: собственные инженерные бригады, контроль качества ISO, соблюдение сроков, фиксированная стоимость.

**Обоснование:** Aetheris показывает 9 преимуществ с детальным описанием. Абис — 6 карточек «Мы обеспечиваем». У Гинт-М преимущества рассредоточены по тексту и не собраны в единый блок, который можно «сканировать глазами».

#### 3.5 Понятные описания услуг вместо перегруженных списков

**Предлагаю:** В карточках услуг оставить 2–3 коротких абзаца: «что делаем», «для кого», «какой результат», а длинные перечни работ вынести на детальную страницу.

**Обоснование:** У TSM услуги описаны простым языком и читаются без перегруза. Это снижает барьер входа и быстрее объясняет ценность.

## 4. ПРОЕКТЫ (/projects/)

### Что сейчас
Фильтры (офисные, спортивные, капстрой, ритейл, жилая, клиенты), сетка 3 колонки, карточка: фото + название + год. Скрытый фильтр по клиентам.

### Мои предложения по изменениям

#### 4.1 Обогатить карточки проектов

**Предлагаю:** Добавить в карточку: площадь, тип работ, эффект при наведении (увеличение + плашка с кратким описанием).

**Обоснование:** Pridex показывает фильтры по площади (6 диапазонов), году, типу проекта и услуге. Карточка Pridex содержит: название, год, адрес, срок реализации, площадь. У Гинт-М только фото + название + год — это слишком мало информации для принятия решения «кликнуть или нет». Площадь — критичный параметр для корпоративного заказчика (он ищет подрядчика под свой масштаб).

#### 4.2 Улучшить фильтрацию

**Предлагаю:** Показывать количество проектов в каждой категории: «Офисные (120) | Спортивные (15)». Добавить фильтр по площади.

**Обоснование:** Pridex предлагает 4 оси фильтрации. Показ количества проектов — это одновременно навигация и социальное доказательство («120 офисных проектов» — это сигнал экспертизы).

#### 4.3 Блок избранных проектов на странице проектов

**Предлагаю:** Вверху — 2–3 «флагманских» проекта крупно, ниже — стандартная сетка.

**Обоснование:** Pridex на главной выделяет избранные проекты (штаб-квартира банка, MR Group). Это привлекает внимание к самым впечатляющим работам и задаёт «планку восприятия» для остальных.

#### 4.4 Фильтр по клиентам — сделать видимым

**Предлагаю:** Вынести логотипы клиентов из скрытого фильтра в видимый блок или превратить в полноценный раздел.

**Обоснование:** Скрытый фильтр = невидимый контент. Если у Гинт-М есть Microsoft, SAP, Kaspersky — эти логотипы должны быть на виду постоянно.

#### 4.5 Сортировка проектов по времени

**Предлагаю:** Добавить переключатель «сначала новые / сначала старые» или сортировку по годам.

**Обоснование:** У TSM есть сортировка по времени — это помогает быстрее находить свежие кейсы и показывает актуальность портфолио.

## 5. СТРАНИЦА ОТДЕЛЬНОГО ПРОЕКТА

### Что сейчас
Хедер с фото, название + год, детали (комплекс работ, площадь, вид работ, адрес), галерея, длинное описание.

### Мои предложения по изменениям

#### 5.1 Структурировать описание

**Предлагаю:** Разбить описание на блоки: Задача → Решение → Результат. Добавить выделенные цитаты/факты.

**Обоснование:** Pridex в карточке проекта даёт: описание + роль Pridex + призыв к действию. Baseform описывает кейсы через «задача → решение → результат». Длинный монолитный текст никто не читает; структура помогает быстро понять суть проекта.

#### 5.2 Добавить навигацию между проектами

**Предлагаю:** «← Предыдущий проект | Следующий проект →» + блок «Похожие проекты» (3–4 карточки из той же категории).

**Обоснование:** Сейчас, дойдя до конца страницы проекта, посетитель упирается в тупик. Нет пути дальше, кроме «назад в каталог». Навигация между проектами увеличивает глубину просмотра и время на сайте.

#### 5.3 Призыв к действию на каждой странице проекта

**Предлагаю:** Финальный блок: «Хотите обсудить похожий проект?» + форма или кнопка.

**Обоснование:** Pridex ставит «Получить консультацию» на каждой карточке проекта. Посетитель, впечатлённый проектом, должен иметь возможность связаться немедленно, без перехода на страницу контактов.

#### 5.4 Отдельный блок «наград» или «отзыв клиента» по проекту

**Предлагаю:** Если проект получал награды или есть отзыв заказчика — вынести короткую цитату и подпись (ФИО, должность, компания) прямо на страницу проекта.

**Обоснование:** У Конте сильные отзывы с конкретными именами. Это резко повышает доверие к конкретному кейсу.

## 6. НОВОСТИ (/event/)

### Что сейчас
Смешанная верстка, карточки разных размеров, нет чётких дат, смешаны новости и анонсы. Текущая система управления сайтом не позволяет менять порядок размещения новостей — доступна только одна фиксированная последовательность.

### Мои предложения по изменениям

#### 6.1 Единообразные карточки

**Предлагаю:** Стандартизировать формат: дата + фото + заголовок + 1–2 строки превью. Добавить категории (награды, мероприятия, проекты, пресса).

**Обоснование:** Pridex имеет аккуратную новостную ленту с датами и блоком «Читайте также». Хаотичная верстка создаёт ощущение неряшливости, что противоречит позиционированию профессионального генподрядчика.

#### 6.2 Подумать о трансформации в блог

**Предлагаю:** Рассмотреть добавление экспертных статей (тренды рынка, кейсы, экспертные комментарии) — по аналогии с блогом Pridex (4 рубрики: Office.Life, Office.Tech, Office.Pro, Office.View).

**Обоснование:** Из 6 референсов (без учёта 3 конкурентов) только Интерьерно активно ведёт блог, и именно у них самый развитый контент-маркетинг. ## 10. РЕКОМЕНДАЦИИ ПО ДИЗАЙНУ

### 10.1 Общая концепция: «Современный технологичный генподрядчик»

Заказчику нравится Pridex. Pridex использует: тёмные секции первого экрана, чистую типографику, много воздуха, крупные фотографии проектов, минимум декоративных элементов, акцент на контент.

**Главный принцип:** дизайн должен транслировать «надёжность + инновационность», а не «мы крупная корпорация из 2005 года».

### 10.2 Цветовая палитра

**Моё предложение: Комбинированная тема (как у Pridex)**

| Элемент | Решение | Обоснование |
|---|---|---|
| Первый экран и футер | Тёмный фон (#0A0A0A – #1A1A1A) | Создаёт премиальное ощущение, как у Pridex |
| Основной контент | Светлый фон (#FFFFFF, #F5F2ED) | Читаемость и лёгкость восприятия |
| Акцент | Один фирменный цвет (терракот, синий или из брендбука Гинт-М) | Для кнопок, ховеров, выделений |
| Текст | #1A1A1A (основной), #6B6B6B (вторичный) | Контрастность и иерархия |

**Почему не полностью тёмная тема:** для генподрядчика со строительным портфолио важно, чтобы фотографии проектов «дышали». На тёмном фоне весь сайт может выглядеть тяжеловесно. Комбинация (тёмный первый экран + светлый контент) даёт баланс.

### 10.3 Типографика

| Элемент | Предложение | Альтернатива |
|---|---|---|
| Заголовки | **Onest** (современный, профессиональный) | Unbounded (более характерный) |
| Текст | **Inter** или **Golos Text** | — |

**Почему не PT Sans:** это базовый шрифт без характера, не создаёт премиального впечатления. Inter — стандарт современного веба, читается отлично на всех экранах.

**Размеры:** H1: 64–80px desktop / 36–48px mobile. Body: 16–18px. Обильные межстрочные интервалы (line-height: 1.5–1.6).

### 10.4 Фотографии

**Предлагаю:**
- Профессиональная фотосъёмка ключевых проектов (если ещё не проведена)
- Единый стиль обработки: нейтральные тона, естественное освещение
- Соотношение сторон: 4:3 или 16:9, object-fit: cover
- WebP-формат для оптимизации

**Обоснование:** Pridex использует крупные, качественные фотографии проектов как главный визуальный инструмент. В строительной отрасли фотография = доказательство качества. Плохие фото обесценивают даже отличный проект.

### 10.5 Анимации

| Тип | Описание | Где применять |
|---|---|---|
| Плавное появление | Блоки появляются при прокрутке | Все секции |
| Поочерёдное появление | Последовательное появление элементов списка | Карточки услуг, проектов |
| Анимация счётчиков | Счётчик цифр при появлении в зоне видимости | Блок цифр |
| Параллакс | Лёгкий параллакс фоновых изображений | Блоки первого экрана |
| Увеличение при наведении | scale(1.02) + плашка | Карточки проектов |
| Из ч/б в цвет | Логотипы клиентов | Блок клиентов |

**Обоснование:** Сейчас сайт полностью статичен. Pridex использует анимации для создания ощущения «живого» сайта. Анимации не должны быть навязчивыми — цель: плавность и отзывчивость, а не «кричащие» эффекты.

### 10.6 Мобильная адаптация

**Предлагаю:**
- Приоритет мобильной версии при разработке
- Бургер-меню только на мобильных (на десктопе — горизонтальная навигация)
- Фиксированная кнопка «Позвонить» или «Написать» на мобильных
- Карточки проектов — в одну колонку
- Элементы, удобные для нажатия (минимум 44px)

**Обоснование:** Bootstrap 3 (текущий фреймворк) устарел и имеет ограничения в адаптивности. Современный генподрядчик должен иметь безупречный мобильный опыт, так как менеджеры часто просматривают сайты подрядчиков с телефона.

### 10.7 Сетка и воздух

**Предлагаю:**
- Контейнер: максимальная ширина 1440px
- 12-колоночная сетка, межколоночные отступы 24–32px
- Отступы между секциями: 80–120px
- Обильный «воздух» (свободное пространство)

**Обоснование:** Pridex использует просторную верстку с большим количеством воздуха. Это создаёт ощущение премиальности и улучшает читаемость. Текущий сайт Гинт-М выглядит «плотным» — больше воздуха = более современное восприятие.

## 12. АДАПТИВНАЯ СЕТКА ПРОЕКТОВ — РЕШЕНИЕ ПРОБЛЕМЫ РАЗНОФОРМАТНЫХ ФОТО

### 12.1 Текущая проблема

Фото проектов от заказчика приходят в разных пропорциях: горизонтальные (16:9, 4:3), вертикальные (3:4, 9:16), квадратные (1:1). На текущем сайте:

- Сетка фиксированная, без адаптивности
- Вертикальные фото обрезаются сверху/снизу, теряя до 40–60% содержимого
- Горизонтальные фото растягиваются или сжимаются
- На мобильных устройствах карточки слишком мелкие или деформированные
- Нет состояний при наведении — невозможно понять, что карточка кликабельна

### 12.2 Три варианта решения

#### Вариант A: Ровная сетка (одинаковые карточки)

Все карточки приводятся к единому соотношению сторон 4:3 через `object-fit: cover`.

**Принцип:** фото заполняет контейнер, выходящие за рамки части обрезаются. Через поле `object-position` (или поле «точка фокуса» в системе управления сайтом) контролируется, какая часть фото видна.

**Breakpoints:**
- Desktop (≥1200px): 3 колонки, gap 20px
- Tablet (768–1199px): 2 колонки, gap 16px
- Mobile (≤767px): 1 колонка, gap 12px

**CSS-основа:**
```css
.project-grid {
  display: grid;
  grid-template-columns: repeat(3, 1fr);
  gap: 20px;
}
.project-card img {
  width: 100%;
  aspect-ratio: 4/3;
  object-fit: cover;
  object-position: var(--focal-point, center);
}
@media (max-width: 1199px) {
  .project-grid { grid-template-columns: repeat(2, 1fr); gap: 16px; }
}
@media (max-width: 767px) {
  .project-grid { grid-template-columns: 1fr; gap: 12px; }
}
```

**Плюсы:** аккуратная сетка, простая реализация, предсказуемая верстка.
**Минусы:** вертикальные фото кадрируются существенно, теряется разнообразие.

#### Вариант B: «Кирпичная» сетка (мой основной вариант)

Каждая карточка сохраняет оригинальные пропорции фото. Колонки выравниваются автоматически.

**Принцип:** через CSS `column-count` или JS-библиотеку (Masonry.js, react-masonry-css) фото размещаются в колонках, каждое сохраняя свои пропорции. Результат — сетка в стиле Pinterest.

**Breakpoints:**
- Desktop (≥1200px): 3 колонки
- Tablet (768–1199px): 2 колонки
- Mobile (≤767px): 1 колонка

**CSS-основа:**
```css
.masonry-grid {
  column-count: 3;
  column-gap: 20px;
}
.masonry-item {
  break-inside: avoid;
  margin-bottom: 20px;
}
.masonry-item img {
  width: 100%;
  height: auto; /* сохраняет пропорции */
  display: block;
}
@media (max-width: 1199px) {
  .masonry-grid { column-count: 2; column-gap: 16px; }
}
@media (max-width: 767px) {
  .masonry-grid { column-count: 1; }
}
```

**Плюсы:** фото показываются полностью, без обрезки; визуально интересная верстка; ощущение «архитектурного журнала».
**Минусы:** неровные края снизу; сложнее в реализации; нужна JS-библиотека для корректного порядка.

#### Вариант C: Смешанная сетка (избранные + обычные карточки)

Топ-проекты крупно (на всю ширину или 2/3 + 1/3), остальные — в uniform grid.

**Принцип:** чередование крупных и мелких блоков создаёт визуальный ритм. Избранные проекты отмечаются флагом в системе управления сайтом.

**Структура:**
```
[  FEATURED 2fr  ] [ SMALL 1fr ]     ← ряд 1: крупный + вертикальный
[ CARD ][ CARD ][ CARD ]              ← ряд 2: три равных 4:3
[      FULL WIDTH PANORAMA      ]     ← ряд 3: панорама на всю ширину
[ CARD ][ CARD ][ CARD ]              ← ряд 4: три равных 4:3
[ SMALL 1fr ] [  FEATURED 2fr  ]     ← ряд 5: зеркально
```

**Плюсы:** флагманские проекты привлекают внимание; журнальная, ритмичная верстка; как у Pridex.
**Минусы:** нужна ручная курация; сложнее в системе управления сайтом; обычные карточки всё ещё кадрируются.

### 12.3 Моё предложение

**Для каталога проектов (/projects/):** я предлагаю Вариант C (Смешанная сетка) — он совмещает аккуратность с визуальным интересом. Флагманские проекты показываются максимально, остальные — в ровной сетке.

**Для галереи внутри проекта:** здесь я вижу оптимальным Вариант B («Кирпичная» сетка) — внутри проекта каждое фото важно и должно показываться полностью.

**Для главной страницы (карусель):** Вариант A (Ровная сетка 4:3) — в карусели все карточки должны быть одного размера для корректной прокрутки.

### 12.4 Поля в системе управления сайтом для каждого фото (обязательно)

Независимо от выбранного варианта сетки, для каждого загруженного фото в системе управления сайтом должны быть поля:

| Поле | Тип | Назначение |
|---|---|---|
| `image` | File | Файл изображения |
| `focal_point` | String (x%, y%) | Точка фокуса для object-position при кадрировании |
| `is_featured` | Boolean | Отметка «показывать крупно» (для смешанной сетки) |
| `alt_text` | String | Описание для поискового продвижения и доступности |
| `sort_order` | Number | Порядок отображения в галерее |

**Поле focal_point** — ключевое решение проблемы обрезки. Когда вертикальное фото кадрируется в 4:3, точка фокуса определяет, какая часть фото остаётся видимой. Например: `object-position: 50% 30%` покажет верхнюю часть фото (лицо человека), а не центр.

### 12.5 Требования к загрузке фото

Для корректной работы адаптивной сетки я предлагаю следующие требования к фото при загрузке в систему управления сайтом:

| Параметр | Требование |
|---|---|
| Минимальное разрешение | 2400px по длинной стороне |
| Формат | JPEG или PNG (автоконвертация в WebP на сервере) |
| Максимальный размер файла | 10 MB (автосжатие на сервере) |
| Цветовой профиль | sRGB |
| Автогенерация размеров | 480w, 768w, 1200w, 1800w (srcset для адаптивных изображений) |

**Серверная обработка (рекомендация):**
- При загрузке автоматически создавать 4 размера для `srcset`
- Конвертировать в WebP с резервом на JPEG
- Извлекать EXIF для определения ориентации
- Автодетекция соотношения сторон для правильного размещения в «кирпичной» сетке
